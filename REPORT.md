# Вывод

По результатам нагрузочного тестирования кэширование улучшило производительность операции `GET /list`.
В режиме без кэша среднее время выполнения `GET /list` составило `6.52 мс`.
В режиме кэша Valkey среднее время выполнения `GET /list` составило `4.77 мс`.
В режиме Client Side Caching среднее время выполнения `GET /list` составило `2.11 мс`. Это лучший результат среди режимов.
Для `POST /list` ускорение не выявлено, потому что операция изменения данных дополнительно выполняет инвалидацию кэша.
Ошибок `500`, `502`, `504` не было.

*Примечание:*

В контроллере скраппера вместо эндпоинта /list эндпоинт /links
